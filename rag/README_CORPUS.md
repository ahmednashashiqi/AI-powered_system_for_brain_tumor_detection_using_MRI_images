# إضافة معلومات جديدة إلى RAG

## أين تضيف النصوص؟

كل النصوص التي يقرؤها الـ RAG موجودة داخل:

```
rag/corpus/
```

يمكنك:
- إضافة ملفات **جديدة** داخل أي مجلد موجود، أو إنشاء مجلد جديد.
- التعديل على الملفات الموجودة.

**المجلدات الحالية:**
- `anatomy/` — ت sequences الرنين (T1, T2, FLAIR, DWI, ADC, SWI)
- `tumor_types/` — glioma, meningioma, pituitary, no_tumor
- `explainability/` — Grad-CAM، XAI، DeepSeek
- `references/` — مصطلحات، وصف الداتاست، مقاييس
- `tips/` — نصائح كتابة التقرير

**صيغ مقبولة:** `.txt` أو `.md` فقط.

---

## شكل الملف

- تكتب **نص عادي** باللغة التي تريد (إنجليزي أو عربي حسب نموذج التضمين).
- يمكن استخدام عناوين وأرقام مثل:
  ```
  ### Glioma – key points
  - T2/FLAIR hyperintense, irregular margins.
  - Post-contrast: ring or heterogeneous enhancement.
  ```
- الطول: أي طول. البرنامج يقطع النص إلى **قطع (chunks)** بحجم ~350 حرف مع تداخل ~100 حرف تلقائياً.

---

## بعد إضافة أو تعديل الملفات

يجب **إعادة بناء فهرس RAG** حتى تُضاف المعلومات الجديدة للبحث:

من مجلد المشروع (KRJAM):

```bash
python rag/build_index.py
```

أو من داخل `rag`:

```bash
cd rag
python build_index.py
```

يمكنك أيضاً حذف الملف `rag/index.npz` وتشغيل التطبيق؛ سيعيد البناء تلقائياً إذا كان الفهرس غير موجود أو أقدم من 7 أيام.

---

## ملاحظات

- **الترميز:** احفظ الملفات بـ **UTF-8** حتى لا يختل النص العربي إن استخدمته.
- **المصدر في النتائج:** اسم الملف والـ chunk يظهران في قسم "REFERENCES (RAG)" في التقرير (مثل `glioma.txt#3`).
- **نموذج التضمين:** الفهرس مبني بـ `BAAI/bge-small-en-v1.5`. للإنجليزي يعطي نتائج جيدة؛ للنصوص العربية يمكن لاحقاً تجربة `BAAI/bge-m3` بتعديل `MODEL_NAME` في `build_index.py` و `retriever.py`.
