About & Methodology

Pipeline, Adaptive RAG, and technology overview

▸ Pipeline (Analysis Flow)

  1. MRI-Gate — The upload is checked to ensure it is a brain MRI (DICOM with Modality=MR, or grayscale PNG/JPEG/TIFF above a minimum size and without strong colour). Non-MRI images are rejected with a clear message.
  2. Classifier — An EfficientNet-B0–based model assigns a label (e.g. glioma, meningioma, pituitary, no_tumor) and a confidence score. This is used to steer RAG and to highlight possible contradictions.
  3. RAG (Retrieval-Augmented Generation) — When enabled and according to the chosen mode, relevant text chunks are retrieved from a medical/neuro corpus and passed as context to the vision–language model.
  4. Vision–Language Model (DeepSeek-VL2) — It describes the image and produces structured Findings, Impression, and Next steps. The report is then formatted for both clinicians and patients.
  5. Consistency & Caution — The system checks for mismatches between the written findings and the classifier, and between RAG sources and the classifier. When found, a short caution is added to the report.

▸ Adaptive RAG

In Adaptive (and Auto) mode, RAG is run only when it is judged useful: when the user asks a custom question, or when the classifier has confidence ≥ 0.60. So the system adapts to the type of query and to the confidence of the detection, instead of always or never using references.

▸ Technology

  • Backend: Flask, PyTorch, DeepSeek-VL2, EfficientNet-B0 classifier, FastEmbed + local RAG index.
  • Auth: Firebase (email, Google, optional phone).
  • Explainability: Grad-CAM overlay when “Advanced Report” is enabled.

This is an assistive tool for education and discussion with clinicians. It does not replace a formal radiology report or clinical decision.
